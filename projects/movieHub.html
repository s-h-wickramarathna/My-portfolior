"movieHUB" - A Movie Trailer Application

Introduction

For my research project, I developed "movieHUB," a movie trailer application designed to give users the ability to explore and watch trailers of various movies. Built using Angular for the frontend and NestJS for the backend, the app focuses on providing a seamless experience for movie enthusiasts to search, view, and interact with trailers.

Technologies Used

  • Frontend: Angular for creating dynamic, responsive UI.
  • Backend: NestJS for managing server-side logic and API services.
  • Database: MySQL for storing trailer data, user information, and more.
  • Async Storage: To handle local storage of user session data on the app.
  • DomSanitizer: This was used in Angular to securely embed YouTube trailers in the app.

Project Overview

"movieHUB" is a movie trailer platform developed as part of my research project using Angular on the client side and NestJS for the backend, with MySQL as the database. The app allows users to register, log in, and browse through a variety of movie trailers embedded via YouTube, with features like advanced search, pagination, and role-based authorization for administrative tasks.

  • User Interface

    The user interface is designed with responsiveness in mind, allowing users to access the platform seamlessly across devices. With a clean layout, users can easily navigate between different sections, search for movies, and watch trailers without hassle.

Key Features

  1. User Registration and Login.

    Users can sign up, log in, and access their personalized dashboard to watch trailers.

  2. Movie Trailer Embedding

    Trailers are embedded using YouTube iframes, with the embedded URLs stored in the MySQL database.

  3. Advanced Search & Pagination

    Users can search for movies by title and browse using pagination, allowing for a more efficient user experience.

  4. Role-Based Authorization

    Only admins have the rights to add new movie trailers, ensuring the platform is well-managed and secure.

Challenges and Solutions

During development, I faced a challenge when trying to display movies by retrieving their URLs from the MySQL database and embedding them in iframes. The issue stemmed from ensuring the URL was sanitized and secure enough to be displayed dynamically in the browser without potential security risks. I resolved this by utilizing Angular's DomSanitizer. Specifically, I used the following import:

import { DomSanitizer, SafeResourceUrl } from '@angular/platform-browser';

This allowed me to sanitize the URL retrieved from the database and safely assign it to the iframe, preventing any XSS (Cross-Site Scripting) vulnerabilities. By leveraging DomSanitizer and SafeResourceUrl, I ensured the application would function securely while embedding movie trailers from YouTube.

Future Enhancements

Although the project met all the requirements, there are several enhancements I would like to implement in the future:

  • JWT Token Authentication

    To enhance the security of user sessions and authorization, I plan to integrate JWT (JSON Web Token) for user authentication in future updates.

  • Real-Time Notifications

    A system for notifying users when new movie trailers are added will be implemented to increase user engagement.

  • Enhanced Role Management

    Expanding the role system to allow more user types such as content moderators for trailer management.

Lessons Learned

  • Handling Embedded Content Securely

    During the development process, I encountered difficulties in embedding external content (YouTube movie trailers) safely into the application. This challenge taught me the importance of securing dynamically loaded content to prevent vulnerabilities such as Cross-Site Scripting (XSS). By using Angular's DomSanitizer, I learned how to properly sanitize and secure external content, ensuring that the iframe could safely display video URLs from the database without compromising the app's security.

  • Understanding the Role of Advanced Search and Pagination

    Implementing advanced search and pagination features in "movieHUB" was a valuable lesson in enhancing user experience. This taught me how important it is to manage large datasets efficiently. I improved my understanding of how to load data in batches, reduce server load, and optimize response times, all while keeping the UI responsive and user-friendly.

  • Role-Based Authorization

    Role-based authorization was a key aspect of the application, where only the admin could add new movie trailers. Implementing this taught me how to manage user roles and access controls effectively in a system. It highlighted the importance of designing a scalable and flexible authorization mechanism that ensures the right users have the correct permissions.

  • Database Management and Integration

    Working with MySQL for storing movie trailers and user data, I gained a deeper understanding of how to structure and optimize relational databases for performance and scalability. Managing the seamless integration of the backend (NestJS) with the database was another key area of learning, especially in terms of querying and data retrieval for movie trailers.

  • Overcoming Development Challenges

    The difficulty of assigning movie URLs from the database to iframes pushed me to improve my problem-solving skills. Researching and implementing a solution using DomSanitizer not only fixed the issue but also provided insight into the complexities of embedding dynamic content within web applications.

  • Building a Full-Stack Application

    Developing "movieHUB" reinforced my skills in building a full-stack application, from designing the frontend with Angular to setting up the backend with NestJS and ensuring smooth communication with MySQL. This experience helped me grow as a full-stack developer, focusing on the synergy between the different layers of the application.
