Building My 1st-Year Final Viver Project: An E-commerce Web Application with PHP and MySQL

Introduction

During my first year at university, I was tasked with developing a comprehensive e-commerce web application for my final project. The objective was to create an online shopping platform from scratch, using pure PHP and MySQL. This project not only challenged my programming skills but also gave me valuable experience in building web applications without relying on frameworks.

Technologies Used

For the tech stack, I relied solely on PHP for the server-side logic and MySQL for database management. PHP handled all the back-end functionality, while MySQL allowed me to manage the product database, user information, and order processing.

Key Features

  1. Speacial Deals

    User can purchase freash, daily & weekly products through the application.

  2. User Authentication

    I implemented a complete user registration and login system. Passwords were securely hashed, and session management allowed users to maintain their login state across different pages.

  3. Product Listing and Details

    Products were dynamically displayed from the MySQL database, with users able to browse, search, and view individual product details on a dedicated page.

  4. Shopping Cart

    A functional shopping cart was one of the highlights of the project. Users could add or remove products from the cart and see real-time updates to the total cost. User could also checkout for their all cart products using PayPal at one time.

  5. Create Seller Accounts

    Local users can create seller accounts by providing their NIC number and images during the registration process. These details are reviewed by the admin, who decides whether to approve or reject the seller request. If the admin approves the request, the user can proceed to set up and manage their shop on the platform.

  6. Admin Login Using OTP System

    Admins can log in using a one-time password (OTP) system. The OTP is generated by the back-end using the uniqueId() function in PHP and is stored in the database before being sent to the admin's registered email. To log in, the admin must first provide a valid email and password. If the credentials are correct, an OTP is sent to the admin's email. The admin then enters the OTP, and if it's correct, they can access the application

  7. Admin Order Confirmation

    Any user can place an order through the application. The admin is responsible for confirming the order before handing over the product to the user. The admin also has the ability to cancel orders. Users can track their orders via the 'Purchase History' page in the application. The admin manages orders by updating the status, including options such as 'Order Confirmed', 'Order Packed', 'Order Dispatched', 'Order Shipped', and 'Order Delivered'.

Challenges and Solutions

  • One of the biggest challenges I faced was managing the shopping cart functionality, particularly ensuring that users could add multiple products and update quantities without disrupting the user experience. When a user increases or decreases the product quantity, the total price updates in real time. Additionally, users should be able to checkout either individual products or multiple products from the cart. I addressed this issue by implementing PHP sessions and a MySQL database. I created a cart table to store the user ID, selected product IDs, and quantities, which allowed me to maintain cart data across different pages seamlessly.

  • Another challenge was handling the database connections and queries efficiently, especially for complex queries involving multiple tables. I learned a lot about optimizing MySQL queries during this process, which helped improve the speed of the platform.

  • Before starting this project, I was unfamiliar with GitHub and how it works. However, toward the end of the project, my lecturer introduced me to GitHub. I then conducted my own research and engaged in self-study to understand its functionality.

Future Enhancements

Although the project met all the requirements, there are several enhancements I would like to implement in the future:

  • Improved User Interface: Currently, the UI is basic. Enhancing the front-end with modern libraries like React or Bootstrap would greatly improve user experience.

  • Advanced Security: Although basic security features like hashed passwords were implemented, there is room to add more advanced security measures, such as using HTTPS and implementing stronger validation checks.

Lessons Learned

This project was a great learning experience, particularly in the areas of server-side programming and database management. Working with PHP without any external frameworks allowed me to gain a deep understanding of how web applications function at their core. I also learned how to structure a MySQL database efficiently and handle CRUD (Create, Read, Update, Delete) operations.
